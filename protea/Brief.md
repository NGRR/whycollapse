# 2. Problema estratégico

La mayoría de las consultoras organizan sus sitios web desde los servicios que venden.

Este enfoque genera tres problemas:

1. Fragmenta el relato.
2. Reduce la diferenciación.
3. Hace que la organización sea percibida como un proveedor más.

Protea requiere posicionarse en una categoría distinta.

No vende servicios aislados.

Desarrolla capacidad adaptativa.

Por ello la arquitectura debe explicar primero el modelo y después los dispositivos.

---

# 3. Tesis central

La arquitectura completa del sitio responde a la siguiente tesis:

> La capacidad de adaptación determina la vigencia de una organización.

Desde esta premisa emerge el modelo Protea:

Complejidad → Adaptabilidad → Innovación

La web debe demostrar que:

* La complejidad es el contexto.
* La adaptabilidad es la capacidad crítica.
* La innovación es una consecuencia.

---

# 11. Principios de diseño

Cada componente debe cumplir simultáneamente:

## Rigor

Debe parecer una herramienta de criterio.

No una campaña publicitaria.

---

## Elegancia

Debe simplificar la complejidad.

No simplificar el contenido.

---

## Profundidad

Debe permitir múltiples niveles de lectura.

---

## Humanidad

Debe representar organizaciones como sistemas vivos.

No como máquinas.

---

## Movimiento

Las animaciones deben sugerir adaptación.

Nunca espectáculo.

---

# 12. Criterio de éxito

La web se considera correctamente diseñada cuando un visitante puede comprender en menos de tres minutos:

1. Qué problema resuelve Protea.
2. Qué es la Inteligencia Adaptativa.
3. Qué es el IAO.
4. Cómo funciona el Sistema Protea.
5. Cuál es el siguiente paso para iniciar una conversación.

Si el visitante comprende únicamente los servicios disponibles, el proyecto ha fallado.

Si comprende el modelo y luego los servicios, el proyecto ha cumplido su objetivo.

# Declaración final

Protea no busca posicionarse como una consultora más.

Busca consolidar una nueva categoría basada en la comprensión de la complejidad, el desarrollo de capacidades adaptativas y la generación de innovación con sentido.

La arquitectura, la experiencia de usuario y el sistema visual han sido diseñados para expresar esa tesis de forma coherente, medible y escalable.
